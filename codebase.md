### 1. Minimalist Philosophy
* **Write only what is necessary:** Do not add boilerplate, speculative features, or "just in case" code. Every line must serve a direct, immediate purpose.
* **Keep functions single-purpose:** Functions should do one thing, do it exceptionally well, and remain short (ideally under 20–30 lines).

### 2. Readability & Clarity with cleverness
* **Prioritize explicit over implicit:** Avoid overly complex one-liners, deeply nested ternaries, or obscure language hacks. Write code that a senior developer can understand at first glance.
* **Self-documenting code:** Use highly descriptive variable and function names (e.g., `calculate_total_price` instead of `calc_tot`). 

### 3. Documentation & Comments
* **Explain the "Why", not the "What":** Do not comment on self-explanatory code. Only use comments to explain complex logic, edge cases, or specific architectural decisions.
* Keep all markdown explanations within `codebase.md` brief, structured, and punchy.
